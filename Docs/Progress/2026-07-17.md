# 2026-07-17 작업 목표

## 오늘 목표 (우선순위 순)

- [x] **1. `COL_HitBox`의 데미지/그로기 로직을 몬스터 자신의 BP로 이관** — `BP_Alatreon`에 `TakeDamage` 이벤트 생성, `Owner`(공격자 액터)를 파라미터로 받아 `Get Ability System Component`로 공격자 ASC 도출 → `Make Outgoing Spec`은 공격자 ASC로, `ApplyGameplayEffectSpecToTarget`은 자기 자신 ASC로 분리. `COL_HitBox`는 캐스트 후 이벤트 호출만 하도록 단순화 완료. **HP 실제 감소 확인 완료.**
- [x] **2. 몬스터 회전 완성** — `FaceTarget`/`FN_Rotator`/`FN_Turn`으로 각도 분기(104/105/107/108) + 백스텝(106, 너무 가까울 때) + 접근(`AI MoveTo`, 너무 멀 때) 전부 완성 및 테스트 완료. Tick 기반 임시 회전은 삭제하고 이 이벤트 기반 시스템으로 완전히 대체함. `ComboStep`이 콤보 시작 시 `Target Actor`를 한 번 캐스트해서 고정하되, 콤보 스텝마다(매 공격 시도) Blackboard에서 다시 fetch하는 구조라 위치/유효성은 매번 최신으로 반영됨.
- [ ] **3. 공격 패턴 3가지 이상** — `AM_Alatreon_Combat_Attack_Stamp` 기존 확보됨(1개). 나머지 2개+는 추가 확보 필요(Blender 613개 클립 중에서 찾아야 할 수도 있음).
- [ ] **4. 그로기 중간 소경직/대경직 모션** — C++ 이벤트(`OnGroggyFlinch`/`OnGroggyDown`)는 이미 있음. 전용 애니메이션 없음 확인됨 → 오늘은 이벤트 배선까지만, 실제 모션은 애셋 나오는 대로 교체.
- [ ] **5. 죽음 모션** — 사망 감지 로직(Health 0 → OnDeath) 자체가 없음, 전용 애니메이션도 없음 → 오늘은 뼈대(이벤트 트리거)만 목표, 실제 모션은 다음 세션 가능성 높음.

## 애셋 확인 결과 (작업 시작 전 조사)

- 104~108 원본 AnimSequence: 임포트 완료 (`CombatTurn` 폴더), 전투용 AnimMontage는 아직 미생성
- 공격 몽타주: `AM_Alatreon_Combat_Attack_Stamp` 1개만 확인됨
- 소경직/대경직/사망 전용 애니메이션: 확인 안 됨 (없을 가능성 높음)

## 진행 상황 / 막힌 지점

- 회전/포지셔닝(104~108, 106 백스텝, 접근 Move To) 완성, 버그 여러 개 수정하며 테스트 완료 (`FN_Turn` Abs 누락, `FN_Turn` 호출 누락, `Target Actor` 세팅 순서, `AI MoveTo` Acceptance Radius/Destination 등)
- 근접 공격 몽타주 5개 확보(스탬프/턴어택/꼬리치기/머리흔들기/돌진), 우선 3개(스탬프/턴어택/머리흔들기)로 랜덤 재생 시도 중
- 공격 판정(히트박스) 설계 완료 — 팀원 `NJWs/COL_SwordAtk` 코드 분석해서 `COL_MonsterAtk`(COL_HitBox 상속) 방식으로 결정. **자세한 설계는 `Docs/Design/Alatreon_Monster_Design.md` 6-1절 참고**
- 넉백은 `LaunchCharacter`로, `COL_HitBox`가 아니라 `BP_PlayerCharacter`의 `Take Damage`에 추가 예정

## 오늘 안 끝나면 다음 세션에 이어할 것

1. `COL_MonsterAtk` 구현 (설계 문서 6-1절대로)
2. `StartMeleeHitbox` 이벤트 + 공격 몽타주에 Anim Notify 배치
3. `PlayAttackStep` 랜덤 3패턴 재생으로 교체
4. 플레이어 `Take Damage`에 넉백(`LaunchCharacter`) 추가
5. 소경직/대경직/사망 모션은 애셋 확보 여부 확인 필요 (아직 목표 4, 5 손 안 댐)
